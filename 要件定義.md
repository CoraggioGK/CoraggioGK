# 法人ホームページ 要件定義書

最終更新: 2026-06-25

## 1. プロジェクト概要

GitHub Pages 上で公開する、IT・サービス業の法人向けコーポレートサイトを制作する。
1ページ完結（ランディングページ型）の構成とし、会社の信頼性とサービス内容を簡潔に伝えることを目的とする。

## 2. 前提・制約条件

| 項目 | 内容 |
|------|------|
| ホスティング | GitHub Pages（静的サイトのみ。サーバーサイド処理は不可） |
| 構成 | 1ページ完結（LP型）／縦スクロール |
| 言語 | 日本語のみ |
| 問い合わせ | 外部フォームサービス連携（例: Formspree / Google フォーム） |
| 動的処理 | DB・サーバー処理は使えないため、フォーム送信は外部サービスに委譲 |

## 3. 技術構成

- **HTML / CSS / JavaScript（バニラ）** — フレームワークなしのシンプル構成。ビルド不要でそのまま GitHub Pages に置ける。
- レスポンシブ対応（PC・タブレット・スマホ）
- 外部依存を最小化（CDN フォント程度。画像は自前 or プレースホルダ）
- SEO 基本対応（title / meta description / OGP / 構造化データ）

> 将来的に Astro 等の静的サイトジェネレータへ移行する選択肢もあるが、初期はメンテのしやすさ優先でバニラ構成とする。

## 4. ページ構成（セクション一覧）

縦1ページに以下のセクションを順に配置する。

1. **ヘッダー / ナビゲーション** — ロゴ＋アンカーリンク（追従固定、スマホはハンバーガー）
2. **ヒーロー（メインビジュアル）** — キャッチコピー＋CTAボタン（問い合わせへ誘導）
3. **サービス紹介** — 提供サービスをカード形式で3〜4点
4. **会社の強み / 特徴** — 選ばれる理由を3点程度
5. **実績 / 導入事例**（任意） — 数字や事例で信頼性を訴求
6. **会社概要** — 社名・所在地・設立・代表者・事業内容などのテーブル
7. **お問い合わせ** — 外部フォーム埋め込み or 連携
8. **フッター** — コピーライト・SNSリンク・プライバシーポリシー等

## 5. 機能要件

| ID | 機能 | 内容 |
|----|------|------|
| F-01 | レスポンシブ表示 | 画面幅に応じてレイアウト最適化 |
| F-02 | スムーススクロール | ナビのアンカークリックで該当セクションへ |
| F-03 | ハンバーガーメニュー | スマホ表示時のナビ開閉 |
| F-04 | 外部問い合わせフォーム | Formspree 等へ送信。送信完了の表示 |
| F-05 | スクロール時の表示演出 | セクションのフェードイン（任意・軽量に） |
| F-06 | OGP / SEO メタ | SNSシェア・検索向けメタ情報 |

## 6. 非機能要件

- **パフォーマンス**: 画像最適化・遅延読み込みで初回表示を軽く保つ
- **アクセシビリティ**: 適切な見出し階層・alt・コントラスト比の確保
- **保守性**: HTML/CSS/JS をファイル分割し、テキスト・色をまとめて変更しやすく
- **ブラウザ対応**: 最新の Chrome / Edge / Safari / Firefox

## 7. デザイン方針（IT・サービス業向け）

- クリーン＆モダン。余白を活かしたミニマルなレイアウト
- アクセントカラー1色＋無彩色ベースで信頼感を演出
- サンセリフ系の読みやすい日本語フォント（Noto Sans JP 等）
- 角丸・軽いシャドウでソフトな印象

## 8. 公開方法（GitHub Pages）

1. GitHub にリポジトリを作成
2. `index.html` などをリポジトリ直下に配置
3. Settings → Pages で公開ブランチ（main / `/root`）を指定
4. `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開
5. （任意）独自ドメインを CNAME で設定

## 9. 今後の確定が必要な情報（コンテンツ）

実装前に以下が決まると、仮テキストを実データに差し替えられる。

- [ ] 会社名・ロゴ
- [ ] キャッチコピー / サービス説明文
- [ ] 提供サービスの具体内容（3〜4点）
- [ ] 会社概要（所在地・設立・代表者など）
- [ ] 問い合わせ先（フォームサービスの送信先 or メールアドレス）
- [ ] ブランドカラー / 参考にしたいサイト

## 10. 進め方の提案

1. 本要件定義の確認・調整
2. デザインのトーン確定（配色・参考サイト）
3. 実装（まずはダミーテキストで全体構築）
4. コンテンツ差し替え
5. GitHub Pages へデプロイ
